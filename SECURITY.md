# 安全策略

Clevo Control Center for Linux 包含内核模块，并会调用 ACPI/DCHU 固件接口。错误写入可能导致系统不稳定、无法启动、散热异常，甚至需要 BIOS/EC 恢复。请不要在非 Clevo 系设备、虚拟机或未验证固件上尝试写入功能。

## 支持范围

安全修复只针对当前 `master` 和最新公开 Release。项目仍处于早期硬件验证阶段，不承诺所有 Clevo 衍生机型都兼容。

## 私密报告漏洞

请优先使用 GitHub 仓库的 **Security Advisories** 创建私密报告。无法使用该功能时，可发送邮件到 `qcqcqc@zust.online`，主题注明 `clevo-control-center security`。

报告中请包含：

- 受影响的提交或版本；
- 可复现步骤和实际影响；
- 是否需要特定机型、能力位或固件版本；
- 已完成的只读诊断和恢复措施；
- 你建议的修复方向（如有）。

请不要在修复发布前公开利用代码，也不要发送机器序列号、账户凭据或未清理的固件映像。维护者确认问题后会协调复现、修复和披露时间；当前项目没有承诺固定响应 SLA。

普通崩溃、UI 问题、打包问题和不涉及安全边界的兼容性报告请使用 GitHub Issues。
